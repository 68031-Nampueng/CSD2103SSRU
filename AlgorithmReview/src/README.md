Design and Analysis of Algorithms
งานทบทวนก่อนเรียน (เรื่อง Array, Stack, Queue ด้วยภาษา Java)

กรณีศึกษาที่ 1 : Array ระบบวิเคราะห์คะแนนสอบก่อนเรียนของนักศึกษา
// ตัวอย่างผลลัพธ์การรันโปรแกรม (Output)
1. คะแนนรวมของนักศึกษาทั้งหมด : 62 คะแนน
2. คะแนนเฉลี่ย : 6.2 คะแนน
3. คะแนนสูงสุด : 10 คะแนน
4. คะแนนต่ำสุด : 2 คะแนน
5. จำนวนนักศึกษาที่ได้คะแนนตั้งแต่ 7 คะแนนขึ้นไป : 5 คน
6. รายชื่อนักศึกษาที่ควรได้รับการทบทวนเพิ่มเติม : นักศึกษาคนที่ 3 (4 คะแนน), นักศึกษาคนที่ 8 (3 คะแนน), นักศึกษาคนที่ 10 (2 คะแนน)

กรณีศึกษาที่ 2: Stack ระบบ Undo ในโปรแกรมจดบันทึก
// ตัวอย่างผลลัพธ์การรันโปรแกรม (Output)
1. สถานะ Stack ปัจจุบัน : [Type Data, Type Structure, Delete Structure, Type Algorithm, Type Java]
2. Undo คำสั่งล่าสุด 2 ครั้ง & 3. แสดงคำสั่งที่ถูก Undo : 
   -> Undo ครั้งที่ 1 ยกเลิกคำสั่ง [Type Java]
   -> Undo ครั้งที่ 2 ยกเลิกคำสั่ง [Type Algorithm]
4. สถานะของ Stack หลังจาก Undo : [Type Data, Type Structure, Delete Structure]
// Stack ใช้หลักการ LIFO อย่างไร 
ในโปรแกรมนี้ Stack ถูกใช้เก็บประวัติคำสั่งของผู้ใช้ เมื่อมีการ Undo โปรแกรมจะใช้ pop() เพื่อนำคำสั่งล่าสุดออกก่อนเสมอ ดังนั้นคำสั่ง [Type Java] และ [Type Algorithm] ซึ่งถูกเพิ่มเข้ามาหลังสุด จึงถูกยกเลิกก่อนคำสั่งอื่น ๆ แสดงให้เห็นถึงหลักการ LIFO

กรณีศึกษาที่ 3: Queue ระบบคิวผู้ป่วยในคลินิกสุขภาพ
// ตัวอย่างผลลัพธ์การรันโปรแกรม (Output)
1. สถานะคิวปัจจุบัน : [P001, P002, P003, P004, P005]
2. เรียกผู้ป่วยเข้ารับบริการ : 
   -> เรียกผู้ป่วย [P001] เข้ารับบริการ
   -> เรียกผู้ป่วย [P002] เข้ารับบริการ
3. ผู้ป่วยที่มาลงทะเบียนเพิ่ม : [P006, P007] 
4. ผู้ป่วยคนถัดไป : [P003]
5. จำนวนผู้ป่วยที่ยังรออยู่ : 5 คน
6. สถานะคิวผู้ป่วยล่าสุด : [P003, P004, P005, P006, P007]
// Queue ใช้หลักการ FIFO อย่างไร
ในโปรแกรมนี้ Queue ถูกใช้จำลองระบบคิวผู้ป่วยของโรงพยาบาล โดยผู้ป่วยจะถูกเพิ่มเข้าท้ายคิวด้วย add() และเรียกเข้ารับบริการจากหน้าคิวด้วย remove() ทำให้ผู้ที่ลงทะเบียนก่อน เช่น P001 และ P002 ได้รับบริการก่อนผู้ที่มาทีหลัง เช่น P006 และ P007 แสดงให้เห็นถึงหลักการ FIFO
